## Description

[Nest](https://github.com/nestjs/nest) framework TypeScript starter repository.

## Project setup

```bash
$ npm install
```

## Compile and run the project

```bash
# development
$ npm run start

# watch mode
$ npm run start:dev

# production mode
$ npm run start:prod
```

## Run tests

```bash
# unit tests
$ npm run test

# e2e tests
$ npm run test:e2e

# test coverage
$ npm run test:cov
```

## Recordings Multipart (MinIO)

El backend soporta subida de grabaciones por chunks con multipart upload firmado para MinIO/S3 compatible, manteniendo el fallback legado por archivo unico.

### Endpoints

- `POST /api/v1/recordings/minio/init`
- `POST /api/v1/recordings/minio/part-url`
- `POST /api/v1/recordings/minio/complete`
- `POST /api/v1/recordings/minio/abort`
- `POST /api/v1/recordings/upload` (legado/fallback)

### Variables de entorno MinIO

```env
# Conexion MinIO/S3
MINIO_ENDPOINT=localhost
MINIO_PORT=9000
MINIO_USE_SSL=false
MINIO_REGION=us-east-1

# Credenciales (nuevas, con fallback a MINIO_ROOT_USER/MINIO_ROOT_PASSWORD)
MINIO_ACCESS_KEY=minioadmin
MINIO_SECRET_KEY=minioadmin

# Buckets
MINIO_RECORDINGS_BUCKET=recordings
MINIO_REPORTS_BUCKET=reports

# URL publica opcional para URLs firmadas de reproduccion
MINIO_PUBLIC_URL=http://localhost:9000

# Multipart
MINIO_MULTIPART_STATE_TTL_SEC=21600
MINIO_PRESIGNED_PART_TTL_SEC=900
```

### Nota de CORS para upload directo a MinIO

Para que el frontend pueda leer `ETag` tras subir cada parte, configura CORS en MinIO para el bucket de grabaciones y expone la cabecera `ETag`.

Ejemplo de CORS sugerido en MinIO/S3:

```json
[
  {
    "AllowedOrigins": ["http://localhost:3000"],
    "AllowedMethods": ["PUT", "POST", "GET", "HEAD"],
    "AllowedHeaders": ["*"],
    "ExposeHeaders": ["ETag", "x-amz-request-id", "x-amz-id-2"],
    "MaxAgeSeconds": 3000
  }
]
```
